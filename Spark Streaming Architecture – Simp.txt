Spark Streaming Architecture – Simple Notes
What is Spark Streaming?

Spark Streaming is a component of Apache Spark that processes real-time data streams. It receives continuous data from sources such as Kafka, Flume, sockets, and files, processes it, and sends the results to storage systems or applications.

3. DStreams (Discretized Streams)

A DStream is the basic abstraction in Spark Streaming.

It divides continuous data into small batches.

Example:

0–5 min   → RDD1
5–10 min  → RDD2
10–15 min → RDD3

A DStream is simply a sequence of RDDs.

6. Fault Tolerance

Spark maintains lineage information for each RDD.

Lineage records how an RDD was created.

If a node fails, Spark recreates lost RDDs using lineage.

It provides fault tolerance through lineage and checkpointing.

Spark Streaming – Short Notes (Save for Revision)
Spark Streaming processes real-time data streams using Apache Spark.
Data comes from sources like Netcat, files, or Apache Kafka.
Spark connects to a source (IP + port or broker) and continuously listens for data.
Core Concept
Continuous stream of data is called a DStream (Discretized Stream).
DStream is divided into micro-batches based on time intervals (e.g., 3–5 minutes).
Each batch becomes an RDD.
Processing Flow
Data is received from source
Stream is converted into DStream
DStream is split into RDDs
Same transformations (map, filter, reduceByKey) are applied on each RDD
Actions generate output
Output
Data is stored in HDFS, databases, files, or sent to downstream systems.
Key Point
Only data ingestion changes (Kafka, file, socket), processing logic remains the same as Spark Core.
Simple Flow
Source → DStream → RDDs → Transformations → Actions → Output
One-Line Definition

Spark Streaming converts continuous data into micro-batches (RDDs) and processes them using Spark Core transformations to produce real-time results. 🚀
